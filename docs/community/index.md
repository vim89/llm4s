## üìö Learning Resources

### Learn About LLMs

New to Large Language Models? These free courses cover the fundamentals:

- **[API Fundamentals](https://github.com/anthropics/courses/tree/master/anthropic_api_fundamentals)** ‚Äî Anthropic course covering SDK basics, model parameters, multimodal prompts, and streaming.
- **[Prompt Engineering Interactive Tutorial](https://github.com/anthropics/courses/tree/master/prompt_engineering_interactive_tutorial)** ‚Äî 9-chapter hands-on tutorial covering core prompt engineering techniques with exercises.
- **[Real World Prompting](https://github.com/anthropics/courses/tree/master/real_world_prompting)** ‚Äî Applying prompt techniques to complex, real-world application scenarios.
- **[Prompt Evaluations](https://github.com/anthropics/courses/tree/master/prompt_evaluations)** ‚Äî Measuring, testing, and improving prompt quality in production systems.
- **[Tool Use Course](https://github.com/anthropics/courses/tree/master/tool_use)** ‚Äî Tool-calling patterns and best practices (directly relevant to the LLM4S tool API).
- **[ChatGPT Prompt Engineering for Developers](https://www.deeplearning.ai/short-courses/chatgpt-prompt-engineering-for-developers/)** ‚Äî Free DeepLearning.AI course by Andrew Ng and Isa Fulford covering LLM fundamentals and practical prompt patterns.
- **[Prompt Engineering Guide](https://www.promptingguide.ai/)** ‚Äî Comprehensive reference for prompt engineering techniques, from zero-shot to chain-of-thought, including recent research.

## ü§ù How to Get Help

### Quick Questions

For quick questions:

1. Search **[existing issues](https://github.com/llm4s/llm4s/issues)**
2. Check **[GitHub Discussions](https://github.com/llm4s/llm4s/discussions)**

### Bug Reports

For bugs:

1. Check **[known issues](https://github.com/llm4s/llm4s/issues)**
2. Create a **[minimal reproduction](https://stackoverflow.com/help/minimal-reproducible-example)**

## Become a Contributor

### Start Here

1. **Read the [Contributing Guide](/reference/contributing)** ‚Äî covers PR requirements, code standards, and the submission process
2. **Pick a [`good first issue`](https://github.com/llm4s/llm4s/labels/good%20first%20issue)** ‚Äî start small, build trust
3. **Comment on the issue** before starting work ‚Äî avoids duplicate effort

### Contribution Areas

- **Tests** - Increase test coverage (great first contribution)
- **Documentation** - Fix errors, improve guides
- **Bug Fixes** - Fix reported issues
- **Features** - After 1-2 merged PRs, take on features
- **Examples** - Add sample applications
